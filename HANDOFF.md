# Handoff: Football Beyond Money — Session Continuation

## Project
Web app at **footballbeyondmoney.uk** — payroll-adjusted football analytics for 6 European leagues.
Repo: `github.com/talfarouriarte-cloud/what-money-cant-buy`. Single-file React 18.2.0 + Recharts 2.12.7 (`index.html`), no build step, GitHub Pages.

## Current state: v3.1, SW v66, 317 i18n keys, ~150KB

### Season management — fully automatic
- `update.py`: `CURRENT_SEASON` always derived from date (Aug+ = new season). Season always advances.
- Wages loaded from `all_wages.json` (single source of truth). Missing teams filled from previous season; promoted teams get league minimum.
- `index.html`: `CUR_SN` derived from latest season key in `data.json` at load time.
- CSV URLs, fixture calendar keys — all derived from `CURRENT_SEASON`. Zero hardcoded season strings.
- `new-season.yml` Action creates a GitHub Issue checklist on August 1st.
- `test_season_transition.py` simulates any date to verify behaviour.

### Annual workflow
1. August: season auto-advances. App processes new matches with previous season wages as approximation. `new-season.yml` creates reminder Issue.
2. September (after window closes): take Capology screenshots → Claude → update `all_wages.json` → commit. All data recalculated with accurate wages.
3. That's it. Everything else is automatic.

### Critical rules
- Always deliver `index.html` + `test.html` (identical)
- Always bump SW version
- Never use global `.replace()` on team names
- `all_wages.json` is the single source of truth for wages
- User works on iPad — all UI must be touch-compatible
